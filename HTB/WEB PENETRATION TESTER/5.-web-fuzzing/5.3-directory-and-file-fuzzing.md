# 5.3 Directory and File Fuzzing

Directory and file fuzzing is used to **discover hidden files, directories, and endpoints** in web applications that are not visible to users. These hidden resources may contain sensitive data, backup files, configuration files, or vulnerable old versions.

Finding these hidden assets is important because they can expose:

* Sensitive information (credentials, configs, logs)
* Outdated or vulnerable files
* Admin panels or development environments
* Hidden functionalities

Even if no direct vulnerability is found, this information helps in understanding the **attack surface** of the application.

***

#### **Wordlists**

Wordlists are used to guess possible file and directory names during fuzzing. They contain common names collected from real-world data.

Popular wordlists:

* `common.txt` → Basic and commonly used
* `directory-list-2.3-medium.txt` → Medium-sized directory list
* `raft-large-directories.txt` → Large directory list
* `big.txt` → Very large list (files + directories)

Common location (Kali/Pwnbox):\
`/usr/share/seclists/`

***

#### **Fuzzing with FFUF**

FFUF uses a wordlist to replace the keyword **FUZZ** in a URL and sends requests to find valid resources.

Example:

```bash
ffuf -w wordlist.txt -u http://target/FUZZ
```

***

#### **Directory Fuzzing**

Used to find hidden directories.

Example:

```bash
ffuf -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -u http://IP:PORT/FUZZ
```

If response shows status like **200, 301**, it indicates a valid directory.

***

#### **File Fuzzing**

Used to find hidden files inside directories.

Common extensions:

* `.php`, `.html`, `.txt`, `.bak`, `.js`

Example:

```bash
ffuf -w /usr/share/seclists/Discovery/Web-Content/common.txt -u http://IP:PORT/FUZZ -e .php,.html,.txt,.bak,.js
```
