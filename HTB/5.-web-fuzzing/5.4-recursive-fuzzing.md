# 5.4 Recursive Fuzzing

Recursive fuzzing is a technique used to **automatically discover nested directories and files** in a web application. Instead of manually testing each folder, the tool explores everything step by step.

The process starts by fuzzing the main directory (web root). When the tool finds a valid directory, it **automatically enters that directory and continues fuzzing inside it**. This process keeps repeating, going deeper into the structure until a limit is reached or no more directories are found. It works like a tree, where each discovered folder becomes a new branch to explore.

Recursive fuzzing is useful because it **saves time, reduces manual effort, and ensures deeper coverage**, helping to find hidden files or directories that might contain vulnerabilities.

In **ffuf**, this can be done using the `-recursion` flag. When enabled, ffuf automatically fuzzes newly discovered directories without needing manual input.

Example:

```bash
ffuf -w wordlist.txt -u http://IP:PORT/FUZZ -e .html -recursion
```

Important options:

* `-recursion-depth` → limits how deep the fuzzing goes
* `-rate` → controls request speed to avoid overloading the server
* `-timeout` → sets request timeout

```bash
ffuf -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -ic -u http://IP:PORT/FUZZ -e .html -recursion -recursion-depth 2 -rate 500
```
