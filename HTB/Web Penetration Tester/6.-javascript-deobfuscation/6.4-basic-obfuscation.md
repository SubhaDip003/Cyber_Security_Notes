# 6.4 Basic Obfuscation

Code obfuscation is the process of making code difficult to read and understand while keeping its functionality unchanged. Instead of writing code manually in a complex way, developers and attackers usually use automated tools that transform normal readable code into a confusing format. This makes it harder for humans to analyze or reverse engineer the code.

For example, a simple JavaScript code like:

```js
console.log('HTB JavaScript Deobfuscation Module');
```

prints a message in the browser console. When we run this in tools like [JSConsole](https://jsconsole.com/), it clearly shows the output. This is called **cleartext code**, which is easy to read and understand.

### Minifying JavaScript Code

Minifying JavaScript means reducing the code size and readability by removing spaces, line breaks, comments, and unnecessary formatting. The code stays functional, but becomes harder to read because everything is placed on one line.

Many tools can help us minify JavaScript code, like [javascript-minifier](https://javascript-minifier.com/). We simply copy our code, and click `Minify`, and we get the minified output on the right:

Example:

<figure><img src="../.gitbook/assets/image (2) (1).png" alt=""><figcaption></figcaption></figure>

For larger programs, minified code becomes a long single line, making it less readable. Minification is commonly used to improve website loading speed and is often saved with the `.min.js` extension.

#### **Packing JavaScript Code**

Packing is a common JavaScript obfuscation method used to make code more confusing and difficult to read while keeping the original functionality the same. Instead of showing the real code directly, the packer converts it into an encoded format and wraps it inside a special function.

A packed JavaScript file often starts with something like:

```js
eval(function(p,a,c,k,e,d){...})
```

This pattern is widely recognized as packed code. The `eval()` function is then used to decode and execute the hidden JavaScript code when the script runs.

For example, a simple code like:

```js
console.log('HTB JavaScript Deobfuscation Module');
```

can be transformed into a long unreadable packed version. Even though the packed code looks complex, when executed in the browser or JavaScript console, it still performs the same task and prints the same output.

<figure><img src="../.gitbook/assets/image (4).png" alt=""><figcaption></figcaption></figure>

A packer works by taking all words, strings, and symbols from the original code and storing them in a list or dictionary. Then it uses the function arguments such as `(p,a,c,k,e,d)` to rebuild the original code step by step during execution. Different packers may use slightly different variable names, but the method is similar.

The main advantage of packing is that it reduces code readability and makes reverse engineering more difficult. This is useful for hiding application logic or protecting scripts from casual copying.

However, packed code is not fully secure. Sometimes important strings or keywords remain visible in clear text, which can give hints about what the code does. Skilled analysts can also unpack and study the original code.
