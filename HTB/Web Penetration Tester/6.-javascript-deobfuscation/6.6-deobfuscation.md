# 6.6 Deobfuscation

Now that we understand how code obfuscation works let's start our learning towards deobfuscation. Just as there are tools to obfuscate code automatically, there are tools to beautify and deobfuscate the code automatically.

***

### Beautify <a href="#beautify" id="beautify"></a>

We see that the current code we have is all written in a single line. This is known as <mark style="color:$primary;">`Minified JavaScript`</mark> code. In order to properly format the code, we need to <mark style="color:$primary;">`Beautify`</mark> our code. The most basic method for doing so is through our <mark style="color:$primary;">`Browser Dev Tools`</mark>.

For example, if we were using Firefox, we can open the browser debugger with <mark style="color:$primary;">`CTRL+SHIFT+Z`</mark>, and then click on our script <mark style="color:$primary;">`secret.js`</mark>. This will show the script in its original formatting, but we can click on the '<mark style="color:$primary;">`{ }`</mark>' button at the bottom, which will <mark style="color:$primary;">`Pretty Print`</mark> the script into its proper JavaScript formatting:

### Deobfuscate <a href="#deobfuscate" id="deobfuscate"></a>

We can find many good online tools to deobfuscate JavaScript code and turn it into something we can understand. One good tool is [UnPacker](https://matthewfl.com/unPacker.html). Let's try copying our above-obfuscated code and run it in UnPacker by clicking the `UnPack` button.

> Tip: Ensure you do not leave any empty lines before the script, as it may affect the deobfuscation process and give inaccurate results.

<figure><img src="../.gitbook/assets/image (3) (1) (1).png" alt=""><figcaption></figcaption></figure>

We can see that this tool does a much better job in deobfuscating the JavaScript code and gave us an output we can understand:


As previously mentioned, the above-used method of obfuscation is `packing`. Another way of `unpacking` such code is to find the `return` value at the end and use `console.log` to print it instead of executing it.

### **Reverse Engineering**

Reverse engineering is the process of analyzing obfuscated code to understand how it works and what it does. When code is heavily obfuscated or encoded, automated tools may not be able to fully decode it, especially if custom obfuscation methods are used.

In such cases, we need to manually study the code, break it down step by step, and identify how it was transformed and how it executes. This helps in understanding the original logic and detecting any hidden or malicious behavior.
