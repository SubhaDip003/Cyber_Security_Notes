# 7.7 SQL Operators

Sometimes, expressions with a single condition are not enough to satisfy the user's requirement. For that, SQL supports [Logical Operators](https://dev.mysql.com/doc/refman/8.0/en/logical-operators.html) to use multiple conditions at once. The most common logical operators are `AND`, `OR`, and `NOT`.

***

### AND Operator <a href="#and-operator" id="and-operator"></a>

The <mark style="color:$info;">`AND`</mark> operator takes in two conditions and returns `true` or `false` based on their evaluation:

```sql
condition1 AND condition2

```

The result of the <mark style="color:$info;">`AND`</mark> operation is `true` if and only if both `condition1` and `condition2` evaluate to `true`:

```bash
mysql> SELECT 1 = 1 AND 'test' = 'test';

+---------------------------+
| 1 = 1 AND 'test' = 'test' |
+---------------------------+
|                         1 |
+---------------------------+
1 row in set (0.00 sec)

mysql> SELECT 1 = 1 AND 'test' = 'abc';

+--------------------------+
| 1 = 1 AND 'test' = 'abc' |
+--------------------------+
|                        0 |
+--------------------------+
1 row in set (0.00 sec)

```

In MySQL terms, any `non-zero` value is considered `true`, and it usually returns the value `1` to signify `true`. `0` is considered `false`. As we can see in the example above, the first query returned `true` as both expressions were evaluated as `true`. However, the second query returned `false` as the second condition `'test' = 'abc'` is `false`.

Example Query:

```bash
MariaDB [users]> SELECT username, password, system FROM logins WHERE username = 'sql' AND system = 'Windows';
+----------+----------+---------+
| username | password | system  |
+----------+----------+---------+
| sql      | sql@123! | Windows |
+----------+----------+---------+
1 row in set (0.168 sec)

MariaDB [users]> SELECT username, password, system FROM logins WHERE username = 'sql' AND system = 'Linux';
Empty set (0.275 sec)

```

***

### OR Operator <a href="#or-operator" id="or-operator"></a>

The <mark style="color:$info;">`OR`</mark> operator takes in two expressions as well, and returns `true` when at least one of them evaluates to `true`:

```bash
mysql> SELECT 1 = 1 OR 'test' = 'abc';

+-------------------------+
| 1 = 1 OR 'test' = 'abc' |
+-------------------------+
|                       1 |
+-------------------------+
1 row in set (0.00 sec)

mysql> SELECT 1 = 2 OR 'test' = 'abc';

+-------------------------+
| 1 = 2 OR 'test' = 'abc' |
+-------------------------+
|                       0 |
+-------------------------+
1 row in set (0.00 sec)

```

The queries above demonstrate how the `OR` operator works. The first query evaluated to `true` as the condition `1 = 1` is `true`. The second query has two `false` conditions, resulting in `false` output.

Example Query:

```bash
MariaDB [users]> SELECT username, password, system FROM logins WHERE username = 'Admin' OR system = 'Linux';
+----------+--------------+---------+
| username | password     | system  |
+----------+--------------+---------+
| Admin    | P@$$w0rd     | Windows |
| root     | root@654321! | Linux   |
| htb      | htb@123456!  | Linux   |
+----------+--------------+---------+
3 rows in set (0.163 sec)

MariaDB [users]> SELECT username, password, system FROM logins WHERE username = 'test' OR system = 'Linux';
+----------+--------------+--------+
| username | password     | system |
+----------+--------------+--------+
| root     | root@654321! | Linux  |
| htb      | htb@123456!  | Linux  |
+----------+--------------+--------+
2 rows in set (0.164 sec)
```

***

### NOT Operator <a href="#not-operator" id="not-operator"></a>

The <mark style="color:$info;">`NOT`</mark> operator simply toggles a `boolean` value 'i.e. `true` is converted to `false` and vice versa':

```bash
mysql> SELECT NOT 1 = 1;

+-----------+
| NOT 1 = 1 |
+-----------+
|         0 |
+-----------+
1 row in set (0.00 sec)

mysql> SELECT NOT 1 = 2;

+-----------+
| NOT 1 = 2 |
+-----------+
|         1 |
+-----------+
1 row in set (0.00 sec)

```

As seen in the examples above, the first query resulted in `false` because it is the inverse of the evaluation of `1 = 1`, which is `true`, so its inverse is `false`. On the other hand, the second query returned `true`, as the inverse of `1 = 2` 'which is `false`' is `true`.

Example Query:

```bash
MariaDB [users]> SELECT username, password, system FROM logins WHERE NOT username = 'test' AND  system = 'Linux';
+----------+--------------+--------+
| username | password     | system |
+----------+--------------+--------+
| root     | root@654321! | Linux  |
| htb      | htb@123456!  | Linux  |
+----------+--------------+--------+
2 rows in set (0.168 sec)

MariaDB [users]> SELECT username, password, system FROM logins WHERE NOT username = 'Admin';
+----------+--------------+---------+
| username | password     | system  |
+----------+--------------+---------+
| root     | root@654321! | Linux   |
| Guest    | qwerty@123!  | Windows |
| htb      | htb@123456!  | Linux   |
| sql      | sql@123!     | Windows |
+----------+--------------+---------+
4 rows in set (0.168 sec)
```

### Multiple Operator Precedence <a href="#multiple-operator-precedence" id="multiple-operator-precedence"></a>

SQL supports various other operations such as addition, division as well as bitwise operations. Thus, a query could have multiple expressions with multiple operations at once. The order of these operations is decided through operator precedence.

Here is a list of common operations and their precedence, as seen in the [MariaDB Documentation](https://mariadb.com/kb/en/operator-precedence/):

* Division (`/`), Multiplication (`*`), and Modulus (`%`)
* Addition (`+`) and subtraction (`-`)
* Comparison (`=`, `>`, `<`, `<=`, `>=`, `!=`, `LIKE`)
* NOT (`!`)
* AND (`&&`)
* OR (`||`)

Operations at the top are evaluated before the ones at the bottom of the list. Let us look at an example:

```sql
SELECT * FROM logins WHERE username != 'tom' AND id > 3 - 2;
```

The query has four operations: <mark style="color:$info;">`!=`</mark>, <mark style="color:$info;">`AND`</mark>, <mark style="color:$info;">`>`</mark>, and <mark style="color:$info;">`-`</mark>. From the operator precedence, we know that subtraction comes first, so it will first evaluate <mark style="color:$info;">`3 - 2`</mark> to <mark style="color:$info;">`1`</mark>:

```bash
MariaDB [users]> SELECT * FROM logins WHERE username != 'root' AND id > 3 - 2;
+----+----------+-------------+---------------------+---------+
| id | username | password    | date_of_joining     | system  |
+----+----------+-------------+---------------------+---------+
|  3 | Guest    | qwerty@123! | 2026-04-28 15:14:34 | Windows |
|  4 | htb      | htb@123456! | 2026-04-28 15:14:34 | Linux   |
|  5 | sql      | sql@123!    | 2026-04-28 15:14:34 | Windows |
+----+----------+-------------+---------------------+---------+
3 rows in set (0.171 sec)

MariaDB [users]> SELECT * FROM logins WHERE username != 'tom' AND id > 3 - 2;
+----+----------+--------------+---------------------+---------+
| id | username | password     | date_of_joining     | system  |
+----+----------+--------------+---------------------+---------+
|  2 | root     | root@654321! | 2026-04-28 15:14:34 | Linux   |
|  3 | Guest    | qwerty@123!  | 2026-04-28 15:14:34 | Windows |
|  4 | htb      | htb@123456!  | 2026-04-28 15:14:34 | Linux   |
|  5 | sql      | sql@123!     | 2026-04-28 15:14:34 | Windows |
+----+----------+--------------+---------------------+---------+
4 rows in set (0.168 sec)
```

```sql
SELECT * FROM logins WHERE username != 'tom' AND id > 1;
```

Next, we have two comparison operations, <mark style="color:$info;">`>`</mark> and <mark style="color:$info;">`!=`</mark>. Both of these are of the same precedence and will be evaluated together. So, it will return all records where username is not `tom`, and all records where the <mark style="color:$info;">`id`</mark> is greater than 1, and then apply <mark style="color:$info;">`AND`</mark> to return all records with both of these conditions:

```bash
MariaDB [users]> SELECT * FROM logins WHERE username != 'tom' AND id > 1;
+----+----------+--------------+---------------------+---------+
| id | username | password     | date_of_joining     | system  |
+----+----------+--------------+---------------------+---------+
|  2 | root     | root@654321! | 2026-04-28 15:14:34 | Linux   |
|  3 | Guest    | qwerty@123!  | 2026-04-28 15:14:34 | Windows |
|  4 | htb      | htb@123456!  | 2026-04-28 15:14:34 | Linux   |
|  5 | sql      | sql@123!     | 2026-04-28 15:14:34 | Windows |
+----+----------+--------------+---------------------+---------+
4 rows in set (0.172 sec)
```
