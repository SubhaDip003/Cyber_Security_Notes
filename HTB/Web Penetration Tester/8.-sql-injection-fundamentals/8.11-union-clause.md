# 8.11 Union Clause

So far, we have only been manipulating the original query to subvert the web application logic and bypass authentication, using the <mark style="color:$info;">`OR`</mark> operator and comments. However, another type of SQL injection is injecting entire SQL queries executed along with the original query. This section will demonstrate this by using the MySQL <mark style="color:$info;">`Union`</mark> clause to do <mark style="color:$info;">`SQL Union Injection`</mark>.

***

### Union <a href="#union" id="union"></a>

Before we start learning about Union Injection, we should first learn more about the SQL Union clause. The [Union](https://dev.mysql.com/doc/refman/8.0/en/union.html) clause is used to combine results from multiple <mark style="color:$info;">`SELECT`</mark> statements. This means that through a <mark style="color:$info;">`UNION`</mark> injection, we will be able to <mark style="color:$info;">`SELECT`</mark> and dump data from all across the DBMS, from multiple tables and databases. Let us try using the <mark style="color:$info;">`UNION`</mark> operator in a sample database. First, let us see the content of the <mark style="color:$info;">`ports`</mark> table:

```bash
mysql> SELECT * FROM ports;

+----------+-----------+
| code     | city      |
+----------+-----------+
| CN SHA   | Shanghai  |
| SG SIN   | Singapore |
| ZZ-21    | Shenzhen  |
+----------+-----------+
3 rows in set (0.00 sec)

```

Next, let us see the output of the <mark style="color:$info;">`ships`</mark> tables:

```bash
mysql> SELECT * FROM ships;

+----------+-----------+
| Ship     | city      |
+----------+-----------+
| Morrison | New York  |
+----------+-----------+
1 rows in set (0.00 sec)

```

Now, let us try to use <mark style="color:$info;">`UNION`</mark> to combine both results:

```bash
mysql> SELECT * FROM ports UNION SELECT * FROM ships;

+----------+-----------+
| code     | city      |
+----------+-----------+
| CN SHA   | Shanghai  |
| SG SIN   | Singapore |
| Morrison | New York  |
| ZZ-21    | Shenzhen  |
+----------+-----------+
4 rows in set (0.00 sec)

```

As we can see, `UNION` combined the output of both <mark style="color:$info;">`SELECT`</mark> statements into one, so entries from the <mark style="color:$info;">`ports`</mark> table and the <mark style="color:$info;">`ships`</mark> table were combined into a single output with four rows. As we can see, some of the rows belong to the <mark style="color:$info;">`ports`</mark> table while others belong to the <mark style="color:$info;">`ships`</mark> table.

> Note: <mark style="color:red;">The data types of the selected columns on all positions should be the same.</mark>
