# 7.3 Types of Databases

Databases, in general, are categorized into <mark style="color:$info;">`Relational Databases`</mark> and <mark style="color:$info;">`Non-Relational Databases`</mark>. Only Relational Databases utilize SQL, while Non-Relational databases utilize a variety of methods for communications.

***

### Relational Databases

A relational database is the most common type of database. It stores data in the form of tables made of rows and columns. It uses a <mark style="color:$info;">**schema**</mark>, which is a structure or design that defines how data will be stored and connected.

For example, a company selling products may store customer details, product details, and order records in separate tables.

#### Tables in Relational Databases

Different tables are used for different types of data, such as:

* **Customers Table** – stores customer name, address, contact details, etc.
* **Products Table** – stores product name, price, quantity, etc.
* **Orders Table** – stores records of which customer bought which product.

#### Keys in Tables

Each table usually contains a **key** that uniquely identifies each record.

* **Primary Key** – Unique ID for each row in a table.
* **Foreign Key** – Used to connect one table with another table.

For example:

* In the **users** table, <mark style="color:$info;">`id`</mark> can be the primary key.
* In the **posts** table, <mark style="color:$info;">`user_id`</mark> can be linked to the `id` in the users table.

This allows the database to connect user information with their posts without repeating the same data.

#### Schema

The relationship between tables in a database is called a **schema**. It defines how tables are linked together.

#### Advantages of Relational Databases

* Fast data retrieval
* Easy to manage structured data
* Reduces duplicate data
* Reliable and accurate
* Supports large amounts of data
* Easy to search using SQL queries

#### Examples of Relational Databases

Common relational database systems include:

* MySQL
* Microsoft Access
* Microsoft SQL Server
* Oracle Database
* PostgreSQL

***

### Non-Relational Databases (NoSQL)

A non-relational database is also called a **NoSQL database**. It does not store data in tables with rows and columns like relational databases.

NoSQL databases usually do not use fixed schemas, primary keys, or table relationships. Instead, they store data in different formats depending on the type of data.

Because of their flexible structure, NoSQL databases are highly scalable and suitable for handling large or unstructured data.

NoSQL databases are a good choice when data changes frequently or does not follow a fixed format.

### Common Types of NoSQL Databases

1. **Key-Value Store**\
   Data is stored as a key and its value.
2. **Document-Based**\
   Data is stored in documents such as JSON or XML format.
3. **Wide-Column Store**\
   Data is stored in columns instead of rows.
4. **Graph Database**\
   Data is stored using nodes and relationships.

### Example of Key-Value Model

```json
{
  "100001": {
    "date": "01-01-2021",
    "content": "Welcome to this web application"
  },
  "100002": {
    "date": "02-01-2021",
    "content": "This is the first post"
  }
}
```

In this example:

* **100001** is the key
* The related data is the value

This is similar to a dictionary in programming languages like Python or PHP.

The most common example of a NoSQL database is `MongoDB`.

NoSQL databases are fast, flexible, and useful for modern web applications with dynamic data.

> Non-relational Databases have a different method for injection, known as NoSQL injections. SQL injections are completely different than NoSQL injections.
