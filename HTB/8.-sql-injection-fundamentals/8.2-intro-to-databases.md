# 8.2 Intro to Databases

Before learning SQL Injection, it is important to understand databases and Structured Query Language (SQL). SQL is the language used to communicate with databases and perform different operations.

Web applications use back-end databases to store important data and information related to the website. This may include website files, images, posts, updates, usernames, passwords, and other user details.

There are different types of databases, and each is used for specific purposes. Earlier, many applications used file-based databases, but they became slow when the amount of data increased.

To solve this problem, Database Management Systems (DBMS) were introduced. DBMS helps store, manage, and access data in a faster and more organized way.

***

### Database Management Systems (DBMS)

A Database Management System (DBMS) is software used to create, define, store, and manage databases. It helps users organize and control data efficiently.

Different types of DBMS have been developed over time, such as file-based databases, Relational DBMS (RDBMS), NoSQL databases, Graph databases, and Key/Value stores.

Users can interact with a DBMS in different ways, such as command-line tools, graphical interfaces, or APIs (Application Programming Interfaces).

DBMS is widely used in banking, finance, education, healthcare, and many other sectors to manage large amounts of data.

#### Important Features of DBMS

| Feature                         | Description                                                                          |
| ------------------------------- | ------------------------------------------------------------------------------------ |
| Concurrency                     | Allows many users to access and use the database at the same time without data loss. |
| Consistency                     | Ensures data remains correct, valid, and accurate during all operations.             |
| Security                        | Provides login authentication and permissions to prevent unauthorized access.        |
| Reliability                     | Supports backup and recovery of data if loss or damage occurs.                       |
| Structured Query Language (SQL) | Uses simple commands to store, retrieve, update, and delete data easily.             |

### Architecture

The diagram explains a **three-tier architecture** used in many web applications.

**Tier I – Client Layer**

This layer includes client-side applications such as websites, mobile apps, or GUI programs. Users interact with the system through actions like login, registration, searching, or commenting. The user data is then sent to the next layer through requests or APIs.

**Tier II – Application Layer / Middleware**

This layer works as the middle part between the client and the database. It receives user requests, processes them, applies business logic, and converts them into a format that the database can understand. It uses drivers or libraries to communicate with the database.

**Tier III – Database Layer**

This layer contains the Database Management System (DBMS). It receives queries from the application layer and performs tasks such as inserting, retrieving, updating, or deleting data. After processing, it sends results or error messages back to the application layer.

#### Additional Note

Sometimes the application server and database server may be hosted on the same machine. However, for large systems with many users and large data, they are usually hosted separately for better performance, security, and scalability.
