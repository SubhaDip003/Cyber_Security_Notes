# 8.11 Union Clause

So far, we have only been manipulating the original query to subvert the web application logic and bypass authentication, using the <mark style="color:$info;">`OR`</mark> operator and comments. However, another type of SQL injection is injecting entire SQL queries executed along with the original query. This section will demonstrate this by using the MySQL <mark style="color:$info;">`Union`</mark> clause to do <mark style="color:$info;">`SQL Union Injection`</mark>.

***

### Union <a href="#union" id="union"></a>

Before we start learning about Union Injection, we should first learn more about the SQL Union clause. The [Union](https://dev.mysql.com/doc/refman/8.0/en/union.html) clause is used to combine results from multiple <mark style="color:$info;">`SELECT`</mark> statements. This means that through a <mark style="color:$info;">`UNION`</mark> injection, we will be able to <mark style="color:$info;">`SELECT`</mark> and dump data from all across the DBMS, from multiple tables and databases. Let us try using the <mark style="color:$info;">`UNION`</mark> operator in a sample database. First, let us see the content of the <mark style="color:$info;">`ports`</mark> table:

```bash
mysql> SELECT * FROM ports;

+----------+-----------+
| code     | city      |
+----------+-----------+
| CN SHA   | Shanghai  |
| SG SIN   | Singapore |
| ZZ-21    | Shenzhen  |
+----------+-----------+
3 rows in set (0.00 sec)

```

Next, let us see the output of the <mark style="color:$info;">`ships`</mark> tables:

```bash
mysql> SELECT * FROM ships;

+----------+-----------+
| Ship     | city      |
+----------+-----------+
| Morrison | New York  |
+----------+-----------+
1 rows in set (0.00 sec)

```

Now, let us try to use <mark style="color:$info;">`UNION`</mark> to combine both results:

```bash
mysql> SELECT * FROM ports UNION SELECT * FROM ships;

+----------+-----------+
| code     | city      |
+----------+-----------+
| CN SHA   | Shanghai  |
| SG SIN   | Singapore |
| Morrison | New York  |
| ZZ-21    | Shenzhen  |
+----------+-----------+
4 rows in set (0.00 sec)

```

As we can see, `UNION` combined the output of both <mark style="color:$info;">`SELECT`</mark> statements into one, so entries from the <mark style="color:$info;">`ports`</mark> table and the <mark style="color:$info;">`ships`</mark> table were combined into a single output with four rows. As we can see, some of the rows belong to the <mark style="color:$info;">`ports`</mark> table while others belong to the <mark style="color:$info;">`ships`</mark> table.

> Note: <mark style="color:$warning;">The data types of the selected columns on all positions should be the same.</mark>

***

### Even Columns <a href="#even-columns" id="even-columns"></a>

A <mark style="color:$info;">`UNION`</mark> statement can only operate on <mark style="color:$info;">`SELECT`</mark> statements with an equal number of columns. For example, if we attempt to <mark style="color:$info;">`UNION`</mark> two queries that have results with a different number of columns, we get the following error:

```bash
mysql> SELECT city FROM ports UNION SELECT * FROM ships;

ERROR 1222 (21000): The used SELECT statements have a different number of columns
```

The above query results in an error, as the first <mark style="color:$info;">`SELECT`</mark> returns one column and the second <mark style="color:$info;">`SELECT`</mark> returns two. Once we have two queries that return the same number of columns, we can use the <mark style="color:$info;">`UNION`</mark> operator to extract data from other tables and databases.

For example, if the query is:

```bash
 SELECT * FROM products WHERE product_id = 'user_input'
```

We can inject a <mark style="color:$info;">`UNION`</mark> query into the input, such that rows from another table are returned:

```bash
SELECT * from products where product_id = '1' UNION SELECT username, password from passwords-- '
```

The above query would return <mark style="color:$info;">`username`</mark> and <mark style="color:$info;">`password`</mark> entries from the <mark style="color:$info;">`passwords`</mark> table, assuming the <mark style="color:$info;">`products`</mark> table has two columns.

***

### Un-even Columns <a href="#un-even-columns" id="un-even-columns"></a>

We will find out that the original query will usually not have the same number of columns as the SQL query we want to execute, so we will have to work around that. For example, suppose we only had one column. In that case, we want to <mark style="color:$info;">`SELECT`</mark>, we can put junk data for the remaining required columns so that the total number of columns we are <mark style="color:$info;">`UNION`</mark>ing with remains the same as the original query.

For example, we can use any string as our junk data, and the query will return the string as its output for that column. If we <mark style="color:$info;">`UNION`</mark> with the string `"`<mark style="color:$info;">`junk`</mark>`"`, the <mark style="color:$info;">`SELECT`</mark> query would be <mark style="color:$info;">`SELECT "junk" from passwords`</mark>, which will always return <mark style="color:$info;">`junk`</mark>. We can also use numbers. For example, the query <mark style="color:$info;">`SELECT 1 from passwords`</mark> will always return <mark style="color:$info;">`1`</mark> as the output.

> Note: When filling other columns with junk data, we must ensure that the data type matches the columns data type, otherwise the query will return an error. For the sake of simplicity, we will use numbers as our junk data, which will also become handy for tracking our payloads positions, as we will discuss later.

> 💡Tip: <mark style="color:violet;">For advanced SQL injection, we may want to simply use '</mark><mark style="color:$info;">NULL</mark><mark style="color:violet;">' to fill other columns, as 'NULL' fits all data types.</mark>

The <mark style="color:$primary;">`products`</mark> table has two columns in the above example, so we have to <mark style="color:$info;">`UNION`</mark> with two columns. If we only wanted to get one column 'e.g. <mark style="color:$info;">`username`</mark>', we have to do <mark style="color:$info;">`username, 2`</mark>, such that we have the same number of columns:

```sql
SELECT * from products where product_id = '1' UNION SELECT username, 2 from passwords
```

If we had more columns in the table of the original query, we have to add more numbers to create the remaining required columns. For example, if the original query used <mark style="color:$info;">`SELECT`</mark> on a table with four columns, our <mark style="color:$info;">`UNION`</mark> injection would be:

```bash
NION SELECT username, 2, 3, 4 from passwords-- '
```

This query would return:

```bash
mysql> SELECT * from products where product_id UNION SELECT username, 2, 3, 4 from passwords-- '

+-----------+-----------+-----------+-----------+
| product_1 | product_2 | product_3 | product_4 |
+-----------+-----------+-----------+-----------+
|   admin   |    2      |    3      |    4      |
+-----------+-----------+-----------+-----------+

```

As we can see, our wanted output of the '<mark style="color:$info;">`UNION SELECT username from passwords`</mark>' query is found at the first column of the second row, while the numbers filled the remaining columns.
